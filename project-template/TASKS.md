# TASKS

このファイルは、VWorkで「今やること」「次にやること」「まだやらないこと」を分けるための共通テンプレートです。

Codexに依頼するときは、ここを見れば現在地が分かる状態にします。

## タスク管理の基本原則

- 1タスクは小さくする
- 1回のCodex依頼で終わる大きさにする
- 実行して確認できる成果物を決める
- まだやらないことも明示する
- 完了したらWORKLOG.mdに結果を残す
- 次の改善を必ず1つ以上残す

## Current Task

今やることを1つだけ書きます。

```text
例:
data/sales.csvを読み、月別売上と上位商品をoutput/sales_summary.mdに出力する。
```

## Good Codex Requests

```text
BUSINESS.md、DESIGN.md、SYSTEM.md、TASKS.mdを読んでください。
Current Taskを実装してください。
出力はoutput/に保存し、実行方法と確認結果をWORKLOG.mdに追記してください。
```

```text
今の実装を壊さずに、Next Improvementsの一番上だけ対応してください。
変更したファイルと確認方法を最後にまとめてください。
```

```text
Backlogを見て、次にやるべき順番を経営効果が高い順に並べ替えてください。
まだ実装はしないでください。
```

## 完了条件

タスク完了時は、次を確認します。

- [ ] 実行できた
- [ ] 出力を確認した
- [ ] WORKLOG.mdに実行コマンドを書いた
- [ ] WORKLOG.mdに結果を書いた
- [ ] 次の改善をTASKS.mdに追加した
