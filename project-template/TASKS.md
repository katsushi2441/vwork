# TASKS

このファイルは、VWorkで「今やること」「次にやること」「まだやらないこと」を分けるための共通テンプレートです。

Codexに依頼するときは、ここを見れば現在地が分かる状態にします。

## タスク管理の基本原則

- 1タスクは小さくする
- 1回のCodex依頼で終わる大きさにする
- 実行して確認できる成果物を決める
- まだやらないことも明示する
- 完了したらWORKLOG.mdに結果を残す
- 次の改善を必ず1つ以上残す

自動化や投稿系のタスクでは、実装タスクと運用タスクを分けます。

- 実装タスク: コード、画面、API、設定サンプルを作る
- 運用タスク: 目標件数、実行間隔、再実行条件、監視方法を決める
- 検証タスク: 実際の出力、投稿文、公開URL、エラー表示を確認する

## Good Codex Requests

```text
BUSINESS.md、RULES.md、SERVERS.md、TASKS.mdを読んでください。
Current Taskを実装してください。
出力はoutput/に保存し、実行方法と確認結果をWORKLOG.mdに追記してください。
```

```text
今の実装を壊さずに、Next Improvementsの一番上だけ対応してください。
変更したファイルと確認方法を最後にまとめてください。
```

```text
Backlogを見て、次にやるべき順番を経営効果が高い順に並べ替えてください。
まだ実装はしないでください。
```

## 完了条件

タスク完了時は、次を確認します。

- [ ] 実行できた
- [ ] 出力を確認した
- [ ] WORKLOG.mdに実行コマンドを書いた
- [ ] WORKLOG.mdに結果を書いた
- [ ] 次の改善をTASKS.mdに追加した
