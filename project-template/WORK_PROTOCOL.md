# WORK_PROTOCOL

この案件でAIと作業するときの共通プロトコルです。Codex、Claude Code、ほかのAIエージェント、手作業のどれでも同じ基準で進めます。

## 目的

AIに「なんとなく任せる」のではなく、目的、成果物、確認方法、記録、次の改善を残しながら、小さく確実に業務改善を進めます。

## 基本フロー

1. `BUSINESS.md` で経営課題と最初の成果物を確認する。
2. `RULES.md` で禁止事項、確認が必要な操作、秘密情報の扱いを確認する。
3. `SERVERS.md` でデータ、PC、サーバー、API、公開先、環境変数の場所を確認する。
4. `TASKS.md` の Current Task だけを実行対象にする。
5. 既存ファイルと入力データを見てから実装する。
6. `data/` から読み、`src/` にコードを書き、`output/` に結果を出す。
7. 実行して結果を確認する。
8. `WORKLOG.md` にコマンド、変更ファイル、出力、確認結果、次の依頼文を残す。
9. 次の改善を `TASKS.md` に追加する。

## 完了条件

「できました」と言えるのは、次を満たしたときだけです。

- 実行コマンドまたは確認手順がある。
- 出力ファイル、画面、投稿、公開URLなどの成果物を確認した。
- 変更内容と確認結果を `WORKLOG.md` に残した。
- エラーや未確認事項があれば隠さず書いた。
- 次に頼むとよいことを `TASKS.md` または `WORKLOG.md` に残した。

## 禁止事項

- 動いていないものを動いたと言う。
- 公開URLを確認せずに公開完了と言う。
- ダミー、サンプル、偽物の画面や動画を本物として扱う。
- 既存データを確認なく上書き、削除、送信する。
- 秘密情報をMarkdownやGit管理ファイルに書く。
- エラーを隠して見た目だけ整える。

## 外部送信・本番反映

次は実行前に人間へ確認します。

- メール送信
- SNS投稿
- YouTubeやCMSへの投稿
- 本番サーバー反映
- 既存データの一括変更
- 顧客や取引先への通知
- 決済や契約に関わる操作

## 証跡

可能な限り、次のような証跡を残します。

- 実行コマンド
- 出力ファイルパス
- 公開URL
- APIレスポンスID
- 画面に表示されたステータス
- コピーされた投稿文
- エラー文
- 変更ファイル一覧
