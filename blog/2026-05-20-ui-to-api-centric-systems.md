# 業務システムの本質は「UI中心」から「API中心」へ

昔、業務システムのUIに強く悩んでいた時期があった。

特に、クラサバ型のGUI中心の業務システムから、Webシステムへ移行していった頃は大変だった。今のようにブラウザで高度な動画やリッチUIが当たり前に動く時代ではなく、Web技術そのものに制約が多かったからだ。

「どうすれば使いやすいUIになるか」
「どうすればクラサバ並みの操作性を実現できるか」

当時はそこが大きなテーマだった。

しかし最近、AI駆動の開発を続ける中で、考え方そのものが変わってきた。

## UIの重要性が相対的に下がる

AI駆動のシステムでは、人間向けUIの重要性が相対的に下がり、代わりに「API」が中心になっていく。

なぜなら、これからのシステムでは、人間が画面を操作するだけではなく、AIエージェント自身がシステムを操作するようになるからだ。

つまり重要なのは、

「人間にとって使いやすい画面」

だけではなく、

「AIが理解しやすく、呼び出しやすく、連携しやすい構造」

になっていること。

業務システムの本質が、UI中心からAPI中心へ移行し始めている。

## 業務機能をAPI化する

例えば、

* 商品登録API
* 在庫取得API
* 動画生成API
* 投稿API
* 顧客分析API

のように、業務機能そのものをAPI化しておくことで、AIエージェントがそれらを組み合わせ、自律的に業務を実行できるようになる。

これは単なる「自動化」ではない。

これまで人間がUIを操作して行っていた業務そのものを、AIがAPI経由で実行する世界への変化だ。

## AIがAPIを使い、APIをAIが改善する

そして面白いのは、そのAPIを使うAI自身が、システム開発にも関わり始めていることだ。

AIにとって使いやすいAPIを設計し、そのAPIをAIが利用し、さらにAIがそのシステム自体を改善していく。

業務システムにとって重要な要素が、「画面設計」から「API設計」へ大きく変わり始めていると感じている。
